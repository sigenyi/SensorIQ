# RECOMMENDED EXPOSURE TIMES (Seconds)
## Wall-Mount X-Ray
- Molars: 0.120 - 0.170
- Pre-molars & Canines: 0.080 - 0.110
- Incisors: 0.070 - 0.090
## Handheld X-Ray
- Molars: 0.27 - 0.50
- Pre-molars & Canines: 0.20 - 0.30
- Incisors: 0.17 - 0.23

# HARDWARE FUNDAMENTALS
- Low kVp (60-65): High contrast; best for Caries detection.
- High kVp (70+): Long gray scale; best for Periodontal bone levels.
- mA: Quantity of photons. High mA reduces "graininess" (noise).

# DIAGNOSTIC RECIPES
- Recipe A (Caries): 60-65 kVp | CLAHE 2.5 | Sharpen Weight 2.0 | Gamma 0.8.
- Recipe B (Endo): Gamma 0.5 (lifts shadows) | Sharpen Sigma 5.0.
- Recipe C (Perio): 70+ kVp | Disable S-Curve | Sharpen Weight 1.0.
- Recipe D (Fracture Hunter): 
    Goal: Visualize hairline cracks.  
    Step 1: Increase S-Curve Scale to 0.85 - 0.90 to re-introduce the "raw" tail data.  
    Step 2: Lower Gamma to 0.40 - 0.50 to darken mid-tones, making the structural breaks in the tooth more apparent.

# TROUBLESHOOTING LOGIC
Issue: Grainy/Pixelated:
- Hardware: Increase Exposure Time (see chart) or check mA (photon quantity).  
- Software: Suggest "Zooming out" to 1:1 or standing 3 feet back if the image is over-magnified.
Issue: Rotation/Flip Not Saving:
- FUSE Check: If using DEXIS, Carestream, Eaglesoft, etc., the Jazz tool settings are ignored. The tech must use the host software's native rotation icons.  
Issue: Normalization Ineffective: 
- If the image looks "flat" despite high percentiles, the S-Curve scale is likely too high (e.g., 0.90), which is overriding the data removal.

## SENSOR AGING & DEGRADATION MECHANISM
THE "SILICON FATIGUE" RULE: Digital sensors lose sensitivity over 5-10 years.
SYMPTOM: An old sensor will produce "Grainy" images even when using the "Recommended Exposure Times" listed above.
SYSTEMIC FIX: If the hardware settings (kV/mAs) are correct according to the chart but the image is still noisy, the AI must diagnose "Sensor Fatigue" and recommend:
A permanent 15-20% increase in exposure time (Hardware).
Activating 2x2 Binning in the Jazz settings (Software) to consolidate the remaining sensitivity.

# INTEGRATION TROUBLESHOOTING
- Issue: Trouble with Image rotation/flip in the final result of the image.
- Logic: 
    1. Check if software is in the FUSE list (e.g., DEXIS, Eaglesoft).
    2. If FUSE: Explain that the host software emulates native sensor logic and overrides the Jazz settings. Direct tech to use the software's built-in rotation icons.
    3. If TWAIN: Check rotation settings in Jazz configurator and Ensure "Save Settings" was clicked in the Jazz configurator.

# ORDER OF OPERATIONS IN THE IMAGE PROCESSING: 
- The processing chain in the software is always Noise Filter > Normalize > Sharpen > Gamma.
- If the image is dark at the start (Noise/Normalize), sharpening it further will only make it look worse.

# MASTER RESET
- If refinement fails, recommend a physical baseline of 70kVp / 7mA / 0.10s to rule out hardware variance.
