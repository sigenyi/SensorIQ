# JAZZ SENSOR GLOBAL CONFIGURATION
Change Sizing: Match the magnitude of each change to how far the image is from acceptable. For fine-tuning a nearly-correct image, use small ~5-10% steps. When the image is clearly off, make ONE decisive, correctly-sized move rather than several timid ones — chronic under-correction wastes the technician's time and erodes trust in the tool. Boundary values are LEGITIMATE and often the correct answer: HP=100, LP=0, Gamma=0.35 or 0.85, CLAHE Regions 2x2 or 8x8, Sharpen Weight 2.00. Do NOT shy from extremes when the diagnosis calls for them. The only hard limit is safety: do not make a jump so large that it risks clipping highlights/shadows or pushing the sensor toward saturation. For images described as "unusable," "completely black," or "completely white," large corrective moves are expected and appropriate.

Anti-Loop / Non-Regression: Before recommending any setting change in a refinement turn, inspect every prior attempt in the session. If a setting was already adjusted in an earlier round, do NOT revert it without strong cause — that produces oscillation loops where the technician ends up back where they started. After two refinement rounds without resolution, STOP repeating the same parameters. Step back, re-diagnose the root cause, and propose a fundamentally different approach — a different escalation tier, a hardware re-check, or a re-analysis of which symptom is actually the underlying problem.

Hardware & Binning Discipline: This tool calibrates POST-CAPTURE SOFTWARE settings. Do NOT reflexively change exposure time or enable 2x2 Binning when a fix is hard to find or when several refinements have already been tried.
- Recommend an exposure-time change ONLY for genuine under-exposure, over-exposure, or sensor saturation, verified against the quick_guide exposure ranges.
- Recommend 2x2 Binning ONLY for a weak X-ray source or a diagnosed case of sensor fatigue (see quick_guide).
- In all other cases, work the software ladder first: AN -> CLAHE -> Gamma -> Sharpening -> Contrast/Brightness.

Contrast & Brightness: Lower priority than the data-preserving software ladder above, and you should generally reach for AN / CLAHE / Gamma / Sharpening before them. Reason: Contrast and Brightness modify linear display values rather than the underlying pixel data, so they "decorate" rather than improve the actual radiograph. HOWEVER, they are LEGITIMATE TOOLS, not banned. Use them when the rest of the ladder has been worked and further adjustment would help — particularly Contrast, which can meaningfully enhance perceived detail and "snap" even on already-optimized data. Recommend them when they would genuinely improve diagnostic perception. Do not treat them as untouchable.

- 2x2 Binning: Combines 4 pixels to 1. Increases sensitivity/brightness; decreases resolution. Use only for weak X-ray sources.
- Resend Last Image: Reprocesses previous capture with current settings.
- Processing Chain: Noise Filter > Normalize/CLAHE > Sharpen > Gamma > Out.

# 1. ADAPTIVE NORMALIZATION & S-CURVE LOGIC
- Enable: Activates global histogram stretching.
- Low Percentile (Shadow Cutoff): Defines the cutoff at the dark end of the histogram. Setting value N PRESERVES data above the Nth percentile of darkness; the darkest N% is removed/clipped.
    - DIRECTION: LOWER N (toward 0) = preserve more shadow detail = darker shadows with full gradation. HIGHER N (toward 5) = clip more shadow data = cleaner shadows but at the cost of dark detail.
    - Typical range: 0-5; Default: 1.
    - Use: Raise LP slightly (e.g. 1 -> 2-3) when the image has visible shadow noise or "muddy" dark areas. Lower LP toward 0-1 when dark detail is being lost or shadows feel crushed.
- High Percentile (Highlight Cutoff): Defines the cutoff at the bright end of the histogram. Setting value N PRESERVES data up to the Nth percentile of brightness; the brightest (100-N)% is removed/clipped.
    - DIRECTION: HIGHER N (toward 100) = preserve more bright detail = brighter image with full highlight gradation. LOWER N (toward 95) = clip more bright data = darker image, appropriate when highlights are GENUINELY blown out.
    - Typical range: 95-100; Default: 99. HP=100 is a legitimate, often-correct value when the goal is maximum brightness preservation — DO NOT shy from pushing to the boundary when the diagnosis calls for it.
    - Use: For a TOO-DARK image or one lacking brightness/whites, RAISE HP toward 100 (e.g. 99 -> 100). For genuinely over-exposed/blown-out highlights (verify against the hardware first), LOWER HP toward 95-97 to clip the bad data.
- S-Curve: Controls how much of the "tail" data trimmed by the Low and High Percentiles (data removed from BOTH the dark and bright ends) is saved back into the image.
    - PRINCIPLE: The HIGHER the S-Curve scale, the more of that removed tail data is preserved, producing more BALANCED, gradual shading across the full tonal range. The LOWER the scale, the more data stays clipped, producing punchier local contrast ("snap").
    - Low Scale (0.10 - 0.30): "Hard Clip." Keeps data removed for higher contrast. Best for standard bone/caries checks.
    - High Scale (0.70 - 0.90): "Soft Clip." Saves more removed tail data for balanced shading. Vital for recovering shadow/highlight detail and for seeing hairline fractures that would otherwise be "clipped".

## CLIPPING & DATA LOSS — RECAP
- HP and LP are CUTOFF POINTS, not "amounts to remove." HP=100 / LP=0 = NO clipping (full data preserved). HP=99 / LP=1 = trim 1% at that end. LOWER HP / HIGHER LP = more aggressive clipping.
- Clipping is one-way data loss: once a percentile clip happens at the AN stage, that detail cannot be recovered later in the pipeline (Gamma / CLAHE / Sharpening operate on what AN passes through).
- AI SAFETY RULE: If the tech complains of MISSING DETAIL or a TOO-DARK image, check first whether HP is set too LOW (clipping bright detail the image needs). If the tech complains of BLOWN-OUT highlights, check whether HP is too HIGH for the hardware exposure (preserving over-exposed pixels that should be clipped) — though the more common root cause of blown-out highlights is hardware (mAs/kVp too high), not software.

# 2. IMAGE ROTATION & INTEGRATION BEHAVIOR
- BEHAVIOR RULE: 
    - TWAIN: Rotation/Flip settings in the Jazz tool WILL affect the final image.
    - FUSE: Rotation/Flip settings in the Jazz tool are DISCARDED; must rotate in host software.

- FUSE INTEGRATIONS (Native - Tool Settings Ignored):
    CDR DICOM, Carestream, Dentrix Ascend, DEXIS, Eaglesoft, Sidexis, Vixwin.

- TWAIN INTEGRATIONS (Tool Settings Accepted):
    XDR, Edge Cloud, Curve Hero, Planmeca Romexis, Oryx, Tigerview, Tracker, iDental, 
    Clio, DTX Studio, SOTA, EzDent-i, Open Dental, Tab32, SOPRO, Mipacs, 
    Denticon XV Capture, Denticon XV Web, CliniView, Dentiray Capture, Imaging XL, 
    Prof. Suni, Xray Vision, SIGMA, PatientGallery, Xelis Dental, Overjet, Aeka, 
    CLASSIC, Archy, Mogo, Acteon, VisionX, OneView, Umbie DentalCare, Harmony, CareStack.

- DEFAULT FOR UNLISTED SOFTWARE: Any imaging software not named in the FUSE list above
    (including the "OTHER" option) is treated as TWAIN by default — Jazz rotation/flip
    settings should take effect. Confirm by applying a rotation in the Jazz Configurator,
    clicking "Save Settings," and checking that it persists in the final image. If it does
    NOT persist, treat that software as FUSE and use the host software's native rotation controls.

# 3. CLAHE (LOCALIZED CONTRAST)
- Clip Limit: Max contrast enhancement. Range [1.50 - 2.50]; Default: 2.00.
- Num Regions X/Y: Grid tiles for local contrast optimization. Range [2 - 8] (i.e. 2x2 up to 8x8); Default: 6x6. A FINER grid (7x7-8x8) sharpens small-scale local detail — best for trabecular bone, PDL space, and perio work. A COARSER grid (2x2-4x4) gives broader, smoother contrast — best for general/whole-image balance. ALWAYS set both Num Regions X and Y whenever CLAHE is adjusted; never leave them implied or unchanged by default.

# 4. IMAGE SHARPENING (EDGE ENHANCEMENT)
- Sigma: Blur radius. Range [2.0 - 10.0]; Default: 2.0.
- Weight: Sharpening strength. Range [0.55 - 2.00]; Default: 0.65. (High weights up to 2.00 are used for aggressive edge work, e.g. the Caries recipe.)

# 5. GAMMA & MANUAL ADJUSTMENTS
- Gamma Level: Mid-tone luminance. Range [.35 - .85]; Default: 0.65. DIRECTION: Lower value = BRIGHTER mid-tones; higher value = DARKER mid-tones. The entire range is below 1.0 and the control is capped at 0.85, so Gamma can brighten strongly but can only darken toward neutral — it cannot fully darken a heavily over-penetrated image on its own.
- Manual Contrast: Range [0.00 - 0.10]; Default: 0.00.
- Manual Brightness: Range [-0.10 - 0.10]; Default: 0.00.

# 6. OUTPUT & TWAIN
- Invert: Toggles radiopaque (white bone) vs radiolucent (black bone).
- TWAIN Multiple: Allows multi-frame acquisition.
- TWAIN Wait: Forces trigger wait between frames.
